# 전투 HUD (Battle HUD)

> **Status**: Designed
> **Author**: systems-designer + user
> **Last Updated**: 2026-03-28
> **Implements Pillar**: 전투 가독성 / 조작 응답성 / 전황 이해

---

## Overview

Battle HUD는 플레이어가 전투 중에 가장 자주 바라보는 UI 레이어다.
웨이브 상태, 생명력, 골드, 스킬 쿨다운, 배치 상태, 보스 경고, 지도자 성장, 보물 공방 진입,
승패 결과를 한 화면에서 읽을 수 있게 해 주며, 동시에 스킬 발동과 전투 중 배치 변경을 위한
직접 입력 진입점이 된다.

이 시스템은 단순한 표시판이 아니라, `WaveDefenseSystem`, `SkillActivationSystem`,
`PlacementSystem`, `EconomyManager`, `LeaderSystem`, `TreasureForgeSystem`, `InputManager`의
사건을 모아 플레이어가 즉시 행동할 수 있게 정리하는 전투 오케스트레이터다.

---

## Player Fantasy

> "전투가 시작되자 화면 위에 웨이브 번호와 목숨이 또렷하게 뜬다.
> 왼쪽에는 스킬 버튼이 쌓여 있고, 오른쪽에는 내가 가진 영웅들이 정렬된다.
> 골드가 모자라면 바로 보이고, 스킬이 준비되면 반짝인다.
> 보스가 나오면 화면이 달라지고, 패배 직전에는 목숨 숫자 하나하나가 심장을 조인다.
> 나는 화면을 읽는 동시에 전황을 통제하고 있다."

플레이어가 원하는 감각:
- **한눈에 읽힘**: 웨이브, 목숨, 골드, 스킬 상태가 즉시 보인다
- **행동 유도**: 지금 눌러야 하는 버튼과 지금 옮겨야 하는 영웅이 명확하다
- **전황 압박**: 보스 경고와 목숨 감소가 UI로도 위협적으로 느껴진다
- **모바일 대응**: 작은 화면에서도 조작과 정보 확인이 동시에 가능하다

## Detailed Design

### 시스템 역할

Battle HUD는 스테이지 씬의 `CanvasLayer` 하위에 존재한다.
씬 로컬 UI이므로 Autoload가 아니다. 전투 씬이 사라지면 HUD도 함께 소멸한다.

**파일 위치**: `src/ui/battle_hud.gd`

핵심 책임:
- 웨이브/목숨/골드/보스 상태 표시
- 지도자 레벨, XP, 재능 포인트 표시
- 스킬 버튼 생성 및 쿨다운 표시
- 영웅 배치 패널 상태 표시
- 보물 수량, 위협 단계, 보물 공방 진입 버튼 표시
- 전투 중 선택 상태, 매각 상태, 배치 안내 상태 관리
- 입력 시스템과 전투 시스템 사이의 UI 계약을 중재
- 승패 오버레이와 결과 요약 표시

### HUD 구성

#### 상단 바

화면 상단에는 전투 핵심 수치를 표시한다.

| 항목 | 표시값 | 연동 시스템 |
|------|------|------------|
| 골드 | `gold` | EconomyManager |
| 목숨 | `lives` | WaveDefenseSystem |
| 웨이브 | `wave_index / total_waves` | WaveDefenseSystem |
| 처치 수 | `kills` | WaveDefenseSystem 또는 Combat 집계 |
| 보물 | `treasure_count` | TreasureForgeSystem |
| 위협 등급 | `threat_tier` | WaveDefenseSystem |
| 보스 체력 | 현재/최대 HP 바 | WaveDefenseSystem + EnemyUnit |

상단 바는 숫자만 보여주는 것이 아니라 상태 전환을 전달한다.
예를 들어 `lives`가 5 이하로 내려가면 경고 색으로 바꾸고, 보스 체력이 활성화되면 별도 띠를 띄운다.

#### 좌측 스킬 바

배치된 영웅마다 하나의 스킬 버튼을 표시한다.

| 항목 | 규칙 |
|------|------|
| 버튼 수 | 현재 배치된 영웅 수만큼 동적 생성 |
| 버튼 크기 | 최소 80×80 게임 픽셀 히트 영역 |
| 표시 정보 | 스킬 아이콘, 남은 쿨다운, 준비 상태 테두리 |
| 비활성 상태 | 쿨다운 중 또는 발동 불가 상태 |
| 정렬 | 슬롯/배치 순서와 일치시키는 것을 기본으로 한다 |

#### 좌상단 지도자 패널

전투 화면 좌상단에는 현재 선택한 지도자의 상태를 보여주는 소형 지휘 패널을 배치한다.

| 항목 | 규칙 |
|------|------|
| 지도자 초상 | 텍스트보다 먼저 읽히는 시각 앵커. 지도자 선택 감각을 유지 |
| 지도자 이름/칭호 | 현재 선택한 지도자 표시 |
| XP 바 | 다음 레벨까지 진행률 표시 |
| 재능 버튼 | 미사용 포인트가 있을 때 강조 |
| 해금 재능 요약 | 1줄 또는 2줄로 축약 표시 |

#### 우측 배치 패널

영웅 카드, 웨이브 시작 버튼, 매각 버튼, 속도 버튼을 배치한다.

| 항목 | 규칙 |
|------|------|
| 영웅 카드 | `HeroUnit` 목록을 기반으로 생성 |
| 매각 버튼 | 선택된 영웅이 있을 때만 노출 |
| 웨이브 시작 버튼 | `WaveDefenseSystem`의 상태에 따라 활성화/비활성화 |
| 속도 버튼 | x1 / x2 / x3 상태를 단계적으로 순환 가능 |
| 오디오 버튼 | 사운드/BGM on/off |
| 보물 공방 버튼 | 전투 중 비활성화, PREP/CLEARED 상태에서만 활성화 |

#### 중앙/오버레이 레이어

- `wave_announce`: 웨이브 시작 문구
- `boss_banner`: 보스 등장 경고
- `combo_display`: 연속 처치 또는 순간 버스트 강조
- `floating_text`: 피해 숫자, 골드 획득, 배치 성공/실패 메시지
- `treasure_banner`: 보물 획득/강화 성공/실패 메시지
- `forge_impact_overlay`: 보물 강화 성공/실패 시 유물 아트와 결과를 크게 띄우는 집중 연출
- `end_screen`: 승리/패배 결과 오버레이
- `treasure_forge_modal`: 보유 보물, 유물 레벨, 성공 확률, 실패 누적 표시

### 정보 우선순위

Battle HUD는 항상 모든 정보를 같은 강도로 보여주지 않는다.
가장 중요한 정보가 맨 앞에 보이도록 우선순위를 둔다.

1. 패배 위기: 목숨, 보스 압박, 경고 연출
2. 즉시 행동 필요: 스킬 준비, 배치 가능 슬롯, 매각 가능 상태
3. 메타 성장 정보: 지도자 포인트, 보물 공방 사용 가능 여부
4. 경제 정보: 골드, 보상, 업그레이드 여력
5. 보조 정보: 처치 수, 누적 피해, 속도 상태, 위협 등급

> **설계 결정**: HUD는 "전부 보여주는 UI"가 아니라 "지금 필요한 것만 먼저 보여주는 UI"다.
> 전투 중 숫자가 많아질수록 플레이어는 판단을 멈춘다.

### 상태 기계

Battle HUD는 외부 전투 상태를 반영하는 UI 상태를 가진다.

#### HUD 상태

```
HIDDEN
  │  (LOADING)
  ▼
PLACEMENT
  │  (placement_ready 또는 전투 시작)
  ▼
WAVES
  │  (victory or defeat)
  ▼
RESULT
```

#### HUD 표시 규칙

| HUD 상태 | 표시 내용 |
|----------|----------|
| HIDDEN | 로딩 스플래시 또는 초기화 메시지 |
| PLACEMENT | 영웅 카드, 배치 힌트, 시작 버튼, 골드 |
| WAVES | 상단 바, 스킬 바, 배치 패널, 보스/경고 오버레이 |
| RESULT | 승패 메시지, 별점, 재시도/다시 플레이 버튼 |

#### 연동 상태 반영

WaveDefenseSystem의 Stage/Wave 상태를 그대로 읽어 HUD 모드로 변환한다.

| WaveDefense 상태 | HUD 반응 |
|------------------|----------|
| LOADING | 입력 대부분 잠금, 로딩만 표시 |
| PLACEMENT | 영웅 카드와 배치 안내 노출 |
| PREP | 웨이브 카운트다운, 시작 버튼 활성화 가능 |
| SPAWNING | 시작 버튼 비활성화, 스폰 진행 강조 |
| ACTIVE | 스킬/배치/매각 중심 전투 UI 노출 |
| CLEARED | 보상 표시, 다음 웨이브 전환 예고 |
| VICTORY | 결과 오버레이, 별점 표시 |
| DEFEAT | 패배 오버레이, 재도전 유도 |

### 입력 계약

Battle HUD는 raw input을 직접 해석하지 않고 `InputManager`의 시그널을 구독한다.

#### 소비 시그널

| 시그널 | HUD 사용처 |
|--------|-----------|
| `tapped(position)` | 버튼, 카드, 슬롯 선택 |
| `drag_started(position, start_position)` | 영웅 카드 드래그 시작 |
| `drag_updated(position, delta)` | 드래그 고스트 위치 갱신 |
| `drag_ended(position, start_position)` | 배치/교체/회수 확정 |

#### HUD 버튼 입력 정책

- 버튼은 터치/클릭 모두 동일한 반응을 가져야 한다
- 입력이 겹칠 때는 HUD가 최우선 소비자가 된다
- 전투 중 버튼 누름과 드래그는 서로 충돌하지 않도록 히트 영역을 분리한다

#### 전투 중 허용 조작

| 조작 | 허용 여부 | 조건 |
|------|----------|------|
| 스킬 발동 | 허용 | `WaveDefenseSystem`이 WAVES 상태일 때 |
| 영웅 재배치 | 허용 | `PlacementSystem`이 배치 잠금을 열었을 때 |
| 매각 | 허용 | 선택된 영웅이 있을 때 |
| 속도 전환 | 허용 | SPAWNING/ACTIVE에서만 |
| 웨이브 시작 | 허용 | PREP/PLACEMENT에서만 |

### 스킬 바 동작

#### 버튼 생성

스킬 바는 배치된 영웅을 기준으로 버튼을 만든다.
영웅이 추가되거나 제거되면 버튼 목록도 갱신한다.

```gdscript
func rebuild_skill_bar() -> void:
    _skill_bar.clear()
    for hero in _placement_system.get_placed_heroes():
        _skill_bar.add_button(hero.unit_id, hero.skill_id, hero.skill_cooldown_remaining)
```

#### 쿨다운 표시

- 쿨다운이 0이면 `ready` 스타일
- 쿨다운이 남아 있으면 숫자 오버레이 표시
- 남은 시간이 1초 미만이면 소수점 대신 올림 정수로 표기
- 스킬 발동 실패 시 이유를 짧게 노출한다

#### 비활성 조건

| 조건 | 처리 |
|------|------|
| 영웅이 배치되지 않음 | 버튼 미표시 |
| 쿨다운 중 | 버튼 차단, 오버레이 표시 |
| 스테이지 비전투 상태 | 버튼 비활성 |
| 영웅이 기절/사용 중 | 버튼 차단 |

### 배치 패널 동작

Battle HUD는 영웅 카드가 단순 목록이 아니라 배치 상태를 드러내는 피드백 레이어라고 본다.

#### 카드 상태

| 상태 | 의미 |
|------|------|
| `AVAILABLE` | 아직 배치되지 않음 |
| `PLACED` | 이미 전장에 있음 |
| `SELECTED` | 현재 선택됨 |
| `DRAGGING` | 이동 중 |
| `LOCKED` | 상황상 선택 불가 |

#### 상호작용

- 카드 탭은 선택 또는 배치 모드 진입으로 이어진다
- 점유 슬롯 선택 시 교체 가능 여부를 색으로 알려준다
- 배치 가능한 슬롯은 녹색, 교체 가능 슬롯은 노란색, 불가 슬롯은 붉은색으로 표시한다

### 경제 정보 표시

Battle HUD는 EconomyManager의 잔고를 즉시 보여준다.

#### 표시 항목

| 항목 | 표시 위치 | 갱신 트리거 |
|------|----------|------------|
| 골드 | 상단 바 | `gold_changed` |
| 강화석 | 보조 배지 또는 결과창 | `stones_changed` |
| 영웅 조각 | 도감/전투 외 보조 정보 | `hero_shards_changed` |

#### 표현 규칙

- 골드가 부족하면 해당 영웅 카드 또는 버튼을 흐리게 표시한다
- 보상 획득 시 숫자가 순간적으로 튀는 피드백을 준다
- 경제 정보는 HUD를 어지럽히지 않도록 전투 핵심 수치보다 우선순위를 낮춘다

---

### 결과 화면

전투 종료 시 HUD는 승리 또는 패배 오버레이를 띄운다.

#### 승리

- 별점
- 남은 목숨
- 클리어 시간
- 획득 골드 및 보상
- 다음 행동 버튼: 다시 플레이 / 다음으로 / 종료

#### 패배

- 도달 웨이브
- 처치 수
- 남은 목숨 0 안내
- 재도전 버튼

> **설계 결정**: 결과 화면은 전투 UI와 분리하되, 전투 내 데이터는 끊기지 않게 받아야 한다.
> 사용자는 결과를 보는 순간 다음 시도를 상상할 수 있어야 한다.

---

### HUD 애니메이션

Battle HUD는 시각적으로 과하지 않지만, 중요한 이벤트는 분명하게 보여준다.

| 이벤트 | 연출 |
|------|------|
| 웨이브 시작 | 중앙 웨이브 배너 짧게 등장 |
| 보스 등장 | 화면 상단 붉은 경고 띠 |
| 목숨 감소 | 상단 목숨 숫자 흔들림 |
| 스킬 준비 완료 | 버튼 테두리 발광 |
| 스킬 실패 | 짧은 진동 + 이유 문구 |
| 클리어 | 금색 팝업 + 별점 |

## Interactions with Other Systems

| 시스템 | 방향 | 연동 내용 |
|--------|------|----------|
| **Wave Defense** | Wave → HUD | `wave_changed`, `lives_changed`, `wave_state_changed`, `boss_spawned`, `stage_cleared`, `stage_failed` 수신 |
| **Skill Activation** | HUD → Skill | 버튼 입력으로 `request_skill_cast()` 호출, 쿨다운 표시 |
| **Hero Placement** | HUD ↔ Placement | 카드 선택, 슬롯 하이라이트, 배치/교체/회수 시각화 |
| **Economy** | Economy → HUD | 골드/강화석/조각 잔고 표시, 부족 상태 경고 |
| **Input/Control** | Input → HUD | 탭/드래그 입력을 버튼/카드/슬롯으로 분기 |
| **Combat** | Combat → HUD | 피해 숫자, 치명타, HP 바 갱신 |
| **Audio/VFX** | HUD → Presentation | 중요한 UI 이벤트에 맞춘 짧은 피드백 연출 |

## Edge Cases

| 상황 | 처리 |
|------|------|
| 배치 중 웨이브가 시작됨 | 배치 상태 유지하되 전투 관련 버튼 우선 노출 |
| 스킬 버튼과 카드가 겹치는 위치 | 스킬 버튼이 우선, 카드 선택은 별도 영역으로 유지 |
| 골드가 부족한데 배치 시도 | 즉시 실패 메시지와 함께 배치 불가 표시 |
| 전투 종료 직후 입력 | 결과 오버레이가 모든 입력을 가로챔 |
| 보스 체력바가 있는데 보스가 사망 | 체력바를 즉시 제거하고 클리어 연출로 전환 |
| 화면 회전/리사이즈 | HUD는 안전 여백을 유지하며 재배치 |
| 모바일 소형 화면 | 우측 패널 축소, 핵심 버튼 우선 유지 |

## Tuning Knobs

| 파라미터 | 기본값 | 안전 범위 | 영향 |
|---------|--------|----------|------|
| `top_hud_height` | 32px | 28~44px | 너무 낮으면 가독성 저하, 높으면 전투 화면 압박 |
| `skill_button_size` | 80px | 64~96px | 낮으면 오조작, 높으면 화면 점유 과다 |
| `panel_width` | 72px | 64~96px | 높을수록 카드 가독성 증가, 전장 가시성 감소 |
| `wave_banner_duration` | 2.0s | 1.2~3.0s | 짧으면 인지 어려움, 길면 흐름 방해 |
| `crit_text_scale` | 1.3x | 1.1~1.5x | 피해 감각 강조 정도 |

## Acceptance Criteria

- [ ] 전투 시작 시 골드, 목숨, 웨이브 수가 HUD에 표시된다
- [ ] 배치된 영웅마다 스킬 버튼이 1개씩 표시된다
- [ ] 쿨다운이 0이 되면 버튼이 ready 상태로 바뀐다
- [ ] 배치 가능한 슬롯과 점유 슬롯이 색으로 구분된다
- [ ] 웨이브 시작/보스 등장/클리어/패배가 각각 다른 HUD 연출로 전달된다
- [ ] 입력은 InputManager 시그널을 통해서만 처리된다
- [ ] 골드 부족, 스킬 실패, 배치 실패가 즉시 읽히는 짧은 메시지로 표시된다
- [ ] 승리/패배 결과 화면이 전투 데이터와 함께 정상적으로 표시된다
