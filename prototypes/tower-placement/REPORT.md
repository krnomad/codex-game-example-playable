## Prototype Report: Tower Placement (풍운지기)

### Hypothesis
"슬롯 기반 타워(영웅) 배치 + 웨이브 방어 루프가 웹 브라우저에서 15분 이상
재미있는가?"

### Approach
순수 HTML5 Canvas + Vanilla JS로 구현. 프레임워크 없음.
- Kingdom Rush 스타일 고정 슬롯 8개 + S자형 경로 1개
- 영웅 4종: 무사(근접), 궁수(원거리), 도사(범위), 무녀(지원/감속)
- 적 4종: 도깨비(일반), 산짐승(빠름), 원귀(중간), 대요괴(보스)
- 5 웨이브 + 골드 경제 + 타워 업그레이드 (3레벨)
- 영웅별 액티브 스킬 1개씩 (쿨다운 기반)

### Result

현재 빌드는 직접 플레이 가능한 상태다.
- 권장 링크: `http://127.0.0.1:4173/prototypes/tower-placement/`
- 체크리스트: `PLAYTEST-CHECKLIST.md`
- 이번 패스 변경점: 브리핑 패널 추가, 버튼 상태 정합성 강화, 가장 앞선 적 우선 타게팅 반영, 상태별 BGM/효과음, 전투 리포트형 결과 화면, 지도자 선택/XP/재능 트리, 월광 지휘실 UI 리프레시, 보물 공방/확률 강화 시스템, 후반 난이도 상향, 지도자 초상화 적용, 유물 전용 일러스트 4종 추가, 강화 성공/실패 전면 오버레이 연출 추가, 적 전투 아트 7종 추가, 전장 길/슬롯/받침대 렌더링 리워크

### Presentation Concept

- **BGM 컨셉**: 달빛 계곡을 지키는 한국 판타지 수호전. 배치 단계는 잔잔하고, 전투는 박동감 있게, 보스와 패배는 더 어둡고 낮게.
- **결과 화면 컨셉**: 점수판이 아니라 "이번 판의 전투 리포트". 누가 가장 많이 딜했고 누가 가장 많이 처치했는지 바로 읽혀야 한다.
- **지도자 컨셉**: 영웅을 배치하는 전술 레이어 위에, 이번 판의 운영 방향을 고르는 전략 레이어를 추가한다. 공격형, 경제형, 스킬형 지도자가 같은 조합의 느낌을 바꿔야 한다.
- **UI 컨셉**: 화면을 단순 HUD가 아니라 "월광 전술실"처럼 보이게 만든다. 전장 바깥 UI는 유리 패널처럼 얇고 선명하게, 전장 안은 더 강한 맥박과 긴장감을 준다.
- **보물 컨셉**: 웨이브와 엘리트/보스 처치로 보물을 얻고, 정비 구간에 공방에서 유물을 확률 강화한다. 안정적인 누적 성장과 한 번 더 걸어보게 만드는 리스크/리워드를 동시에 준다.
- **비주얼 패스 컨셉**: 지도자/영웅/유물을 모두 텍스트 위주 정보판이 아니라 "선택하고 싶은 아트 오브젝트"처럼 보이게 만든다. 특히 강화 순간은 결과를 읽는 것이 아니라 체감하도록 연출한다.
- **전장 비주얼 컨셉**: HUD와 모달만 예쁜 게임이 아니라, 실제 전투판 자체가 달빛 전장과 행군로, 제단, 적 실루엣으로 기억에 남아야 한다.

관찰 예정 항목:
- 배치 슬롯이 전략적 의미를 가지는가
- 웨이브 간 자원 관리가 의미 있는 선택인가
- 영웅 간 역할 차이가 체감되는가
- 스킬 발동 타이밍이 긴장감을 만드는가
- 5웨이브 완료까지 재미가 유지되는가

### Metrics
- Frame time: 플레이테스트 시 기록 예정
- Feel assessment: 첫 피드백 수집 대기
- Session length: 첫 외부 플레이 세션 대기

### Recommendation: 플레이테스트 피드백 수집 후 결정

### If Proceeding
- Godot 4로 재구현 (프로토타입 코드 재사용하지 않음)
- 픽셀아트 에셋 교체 (현재 이모지 → 32x32 스프라이트)
- 사운드/음악 추가
- 세이브 시스템 (LocalStorage/IndexedDB)
- 밸런싱 패스 (적 HP, 타워 데미지, 골드 경제)
- 영웅 수집/성장 메타게임 레이어

### Lessons Learned

- 첫 피드백 루프를 빠르게 돌리기 위해 인게임 브리핑 패널과 별도 체크리스트 문서를 추가했다
- 전략 감각 검증의 핵심은 "가장 앞선 적 우선 타게팅"이 실제로 슬롯 가치를 만들고 있는지 확인하는 것이다
