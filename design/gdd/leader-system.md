# 지도자 시스템 (Leader System)

> **Status**: Designed
> **Author**: systems-designer + user
> **Last Updated**: 2026-03-28
> **Implements Pillar**: 플레이 스타일 분화 / 장기 성장 / 선택의 무게

---

## Overview

지도자 시스템은 플레이어가 전투에 들어가기 전에 **한 명의 지도자**를 선택하고,
그 지도자의 고유 특성, 스킬트리, 성장 방향에 따라 같은 영웅 조합이라도 다른 운영을 만들게 하는 메타 시스템이다.

기존의 영웅(무사, 궁수, 도사, 무녀 등)은 전장에 배치되는 전술 단위이고, 지도자는 그 전술 전체를 이끄는 **전략 레이어의 주체**다.
즉, 영웅이 "어떻게 싸우는가"를 담당한다면 지도자는 "무엇을 더 잘하게 만들 것인가"를 결정한다.

이 시스템의 목표는 세 가지다.
- 매 판 시작 시 플레이 스타일을 선택하는 재미를 만든다
- 전투 후 획득한 경험치를 다음 선택과 성장으로 연결한다
- 결과 화면에서 "이번 판을 이끈 지도자"가 명확히 기억되게 만든다

---

## Player Fantasy

> "이번 판은 누굴 앞세워 싸울까?"
> "나는 공격형 지도자로 빠르게 밀어붙일 수도 있고, 수비형 지도자로 마지막까지 버틸 수도 있다."
> "전투를 끝내면 경험치가 쌓이고, 다음엔 새로운 특성이나 스킬 분기를 열 수 있다."
> "같은 영웅을 써도 지도자에 따라 운영이 달라진다."

플레이어가 원하는 감각:
- **선택의 무게**: 전투 전 지도자 선택이 빌드의 방향을 바꾼다
- **성장 누적감**: 한 판의 성과가 XP로 쌓여 다음 해금으로 이어진다
- **역할 분화**: 지도자마다 전투 성향과 경제 성향이 명확히 다르다
- **재도전 동기**: 다른 지도자로 같은 스테이지를 다시 돌고 싶어진다

---

## System Goals

### 반드시 해결할 것
- 전투 시작 전, 플레이어가 지도자 1명을 고를 수 있어야 한다
- 지도자는 단순 스킨이 아니라 고유 특성과 성장 축을 가져야 한다
- 전투 결과에 따라 지도자 XP를 얻고, 레벨업으로 선택지가 열린다
- 레벨업 시 3갈래 스킬트리 중 하나를 선택하는 구조여야 한다
- 지도자 성장 내용이 전투/경제/스킬 운영에 실제 영향을 줘야 한다

### 피해야 할 것
- 영웅 시스템과 중복되는 “또 하나의 캐릭터 수집 UI”가 되는 것
- 스킬트리가 너무 복잡해져 첫 판부터 이해를 막는 것
- 성장 보상이 전투 체감으로 이어지지 않는 것
- 레벨업이 숫자만 오르고 선택이 없는 수동적 메타가 되는 것

## Core Loop

### 1. 전투 전 선택
플레이어는 스테이지 시작 전에 지도자 목록 중 하나를 고른다.
선택한 지도자는 이번 판 전체에 적용된다.

### 2. 전투 중 운영
지도자의 고유 특성과 스킬트리 효과가 전투에 반영된다.
예를 들면 골드 수급, 스킬 쿨다운, 영웅 공격 성향, 배치 보너스 등이 달라진다.

### 3. 전투 후 성장
클리어, 처치, 누적 피해, 생존 시간, 보너스 목표 달성 등으로 지도자 XP를 얻는다.

### 4. 레벨업 및 분기
XP가 누적되면 지도자 레벨이 오르고, 정해진 지점에서 3갈래 스킬트리 중 하나를 선택한다.
선택은 되돌리기 어렵거나, 제한된 재분배 비용을 요구한다.

### 5. 다음 판 재적용
성장한 지도자가 다음 스테이지/웨이브에 영향을 주며, 다른 지도자도 써보고 싶게 만든다.

## Leader Structure

### 지도자 정의

지도자는 다음 요소로 구성된다.
- **이름 / 칭호**
- **정체성 태그**: 공격형, 수비형, 경제형, 전술형 등
- **고유 특성 2개**
- **액티브 또는 패시브 핵심 능력 1개**
- **스킬트리 3갈래**
- **레벨과 XP**

### 지도자와 영웅의 관계

- 지도자는 전장에 직접 배치되는 유닛이 아니다
- 지도자는 전투 전체에 적용되는 오라, 보너스, 규칙 변형을 제공한다
- 영웅은 지도자의 영향을 받지만, 개별 영웅 고유 스킬은 그대로 유지된다
- 지도자 선택은 영웅 구성의 “해석”을 바꾸는 상위 레이어다

> 설계 원칙: 지도자는 영웅을 대체하지 않는다. 지도자는 영웅 조합이 어떻게 작동하는지를 바꾼다.

## XP and Progression

### XP 획득 원천

지도자 XP는 전투 결과를 기반으로 누적된다.

| 획득 원천 | 예시 |
|------|------|
| 웨이브 클리어 | 웨이브 종료 시 기본 XP 지급 |
| 적 처치 | 적 수와 등급에 따라 추가 XP |
| 보스 처치 | 큰 보상 |
| 생존 보너스 | 목숨을 많이 남길수록 보너스 |
| 경제 보너스 | 남은 골드, 효율적 운영 보너스 |
| 스킬 활용 | 지도자 스킬 또는 핵심 시너지 활용 보상 |

### XP 설계 원칙

- 초반 레벨은 빠르게 오른다
- 중반부터는 분기 선택이 의미 있게 느껴져야 한다
- 후반 레벨은 장기 동기용으로 완만한 성장 곡선을 따른다
- 경험치 획득은 “잘했다”는 피드백이어야 하며, 노가다 감각이 너무 강하면 안 된다

### 권장 구조

| 레벨 구간 | 목적 |
|------|------|
| Lv.1-3 | 핵심 특성 학습 |
| Lv.4-6 | 1차 분기 선택 |
| Lv.7-9 | 빌드 강화 |
| Lv.10+ | 후반 최적화 및 마스터리 |

## Skill Tree

지도자마다 스킬트리는 항상 3갈래로 구성된다.
각 갈래는 플레이 스타일을 대표하며, 같은 지도자라도 운영이 달라지게 만든다.

### 공통 트리 예시 구조

1. **전투 트리**
   - 직접적인 전투 효율 상승
   - 공격력, 스킬 피해, 쿨다운, 타격 범위, 연쇄 효과 강화

2. **전술 트리**
   - 배치/운영/상황 대응 강화
   - 고지대 보너스, 타겟 우선순위, 군중 제어, 스킬 조건 완화

3. **국정 트리**
   - 경제/자원/보상 강화
   - 골드, XP, 웨이브 보너스, 업그레이드 비용 감소

### 스킬트리 규칙

- 각 레벨업마다 3개 중 1개만 선택하는 구조를 기본으로 한다
- 같은 갈래를 연속으로 찍어도 되고, 혼합 빌드도 가능하다
- 특정 노드에서는 다음 단계 노드가 열리는 식의 선형 의존성을 둔다
- 최상위 노드는 지도자의 정체성을 확실히 보여주는 “시그니처 노드”여야 한다

### 선택 압박 설계

스킬트리는 단순한 버프 목록이 아니라, 다음 질문을 강요해야 한다.
- 이번 판은 빨리 끝내는 공격형인가
- 안정적으로 버티는 수비형인가
- 자원 효율을 극대화하는 경제형인가

## Leader Themes

아래는 MVP 기준의 지도자 콘셉트 예시다.

### 1. 무장형 지도자
- 방향: 공격 / 근접 / 전방 압박
- 특성: 근접 영웅 강화, 처치 시 가속, 보스전 폭딜
- 어울리는 유저: 적극적으로 밀어붙이고 싶은 플레이어

### 2. 도술형 지도자
- 방향: 범위 / 제어 / 연쇄 효과
- 특성: 광역 피해 증가, 상태 이상 연장, 다수전 효율
- 어울리는 유저: 패턴 읽기와 스킬 타이밍을 좋아하는 플레이어

### 3. 수호형 지도자
- 방향: 방어 / 생존 / 안정성
- 특성: 목숨 보너스, 치유/감속 강화, 방어적 스킬 효율
- 어울리는 유저: 안정적으로 밀고 가는 플레이어

### 4. 상단형 지도자
- 방향: 경제 / 성장 / 장기 운영
- 특성: 골드 수급, 업그레이드 할인, XP 보너스
- 어울리는 유저: 계산과 최적화를 좋아하는 플레이어

> MVP에서는 지도자 3명 정도만 넣어도 충분하다. 종류가 많기보다 차이가 명확해야 한다.

## Connection to Combat

### 전투에 주는 영향

지도자의 효과는 전투의 몇 가지 축에만 명확히 반영되어야 한다.

| 전투 축 | 가능한 영향 |
|------|------|
| 배치 | 특정 슬롯/고지대 보너스, 배치 비용 감소 |
| 공격 | 특정 타입 영웅 공격력/공속 증가 |
| 스킬 | 쿨다운 감소, 범위 증가, 효과 증폭 |
| 생존 | 목숨 보너스, 방어 버프, 회복 증가 |
| 적 대응 | 보스 피해 보너스, 다수 적 처리 보너스 |

### 전투 피드백

- 지도자 특성이 발동하면 별도 색상의 텍스트나 오라를 보여준다
- 결과 화면에는 “이번 판의 지도자 성과”가 표시된다
- 특정 지도자 빌드가 강할 때, 플레이어가 그 이유를 직관적으로 이해해야 한다

## Connection to Economy

지도자 시스템은 경제를 망가뜨리지 않고, 경제의 방향을 바꾸는 역할을 한다.

### 경제 영향 예시

- 골드 획득량 증가
- 업그레이드 비용 할인
- 웨이브 종료 보너스 강화
- 미사용 골드 일부를 XP로 환산
- 특정 조건에서 추가 보상 지급

### 경제 설계 원칙

- 경제형 지도자가 항상 정답이 되면 안 된다
- 공격형/수비형 지도자도 충분한 보상을 받아야 한다
- 경제 보너스는 “더 많은 선택”을 주는 방향이어야지, 전투 난이도를 무력화하면 안 된다

## Connection to Skills

지도자 스킬은 영웅 스킬과 분리된 상위 시스템이다.

### 구분 원칙

- **영웅 스킬**: 전장 단위의 즉시 발동 능력
- **지도자 스킬**: 전투 전체를 바꾸는 진형/오라/전술 개입

### 예시

- 영웅 스킬이 “광역 번개”라면, 지도자 스킬은 “다음 10초간 모든 광역 스킬 피해 +20%”일 수 있다
- 영웅 스킬이 “방벽 생성”이라면, 지도자 스킬은 “방벽 지속시간 증가 및 유지 중 골드 보너스”일 수 있다

### 권장 운영 방식

- 지도자 스킬은 긴 쿨다운을 가진 강한 전술 카드로 둔다
- 영웅 스킬은 자주 쓰는 전투 카드로 둔다
- 두 시스템이 서로 쿨다운과 발동 조건을 공유하면, 플레이어가 “지금 뭘 써야 하나”를 더 잘 판단하게 된다

## MVP Scope

### MVP에 포함
- 지도자 3명
- 전투 전 지도자 선택 화면
- 지도자 고유 특성 1~2개
- 지도자 XP 획득
- 레벨업
- 3갈래 스킬트리
- 전투 HUD에서 지도자 레벨, XP 바, 재능 포인트 표시
- 결과 화면에서 지도자 레벨, XP, 최고 성과 표시
- 지도자 선택에 따른 전투/경제/스킬 차이 체감

### 현재 프로토타입 기준 구현 메모

- 구현 지도자:
  - 태군: 전체 화력/근접/보스 딜 강화
  - 연화: 고지대/웨이브 보너스/원거리 운영 강화
  - 해령: 스킬 쿨다운/스킬 피해/사거리 강화
- XP 획득원:
  - 적 처치
  - 스킬 사용
  - 웨이브 클리어
- 현재 레벨 상한:
  - Lv.4 (초기 1 + 3회 레벨업)
- 현재 UX:
  - 시작 오버레이에서 1명 선택
  - 좌상단 HUD에서 XP/레벨/해금 재능 확인
  - 재능 버튼으로 트리 모달 진입

### MVP에 제외
- 다수 지도자 동시 편성
- 지도자 간 조합 시너지
- 지도자 전용 장비/룬/유물
- PvP 랭킹
- 리셋 후 계승하는 로드 같은 대규모 메타
- 복잡한 스토리 분기

### MVP 성공 기준
- 지도자 선택만으로 플레이 감각이 달라져야 한다
- 플레이어가 “다음 판엔 다른 지도자를 해보고 싶다”라고 느껴야 한다
- 결과 화면에서 성장 이유가 명확히 보여야 한다

## UI and UX Requirements

### 선택 화면
- 지도자 카드형 선택 UI
- 각 카드에는 이름, 역할 태그, 핵심 특성, 추천 플레이 스타일을 보여준다
- 선택 시 전용 프레임과 간단한 애니메이션을 준다

### 전투 HUD
- 현재 선택한 지도자 아이콘 표시
- 지도자 레벨 및 XP 바 표시
- 특성 발동 시각화
- 스킬트리 노드 진행 상태 표시

### 결과 화면
- 이번 판의 점수뿐 아니라 지도자 XP 획득량, 레벨업 여부, 최고 기여 항목을 보여준다
- 최고 데미지 직업처럼, 최고 기여 지도자 특성도 함께 요약할 수 있다

## Content Expansion Path

### 후속 확장 1: 지도자 고유 스킬 추가
- 전투 중 1회 또는 제한 횟수로 쓰는 궁극기
- 현재 MVP보다 훨씬 큰 체감 제공

### 후속 확장 2: 지도자 특성 조합
- 특정 영웅 타입과 지도자 조합 시 추가 시너지
- 예: 무녀형 지도자 + 지원 영웅 조합 강화

### 후속 확장 3: 지도자 숙련도
- 같은 지도자를 많이 쓰면 숙련도 보너스가 붙는 시스템
- 빌드 다양성과 장기 목표를 강화

### 후속 확장 4: 리셋/전직
- 일정 레벨에서 전직 선택
- 완전히 다른 플레이 스타일로 변화

### 후속 확장 5: 스토리형 지도자 서사
- 각 지도자별 챕터, 대사, 관계도, 해금 조건 추가

## Risks

### 리스크 1: 선택 피로
지도자와 영웅 선택이 동시에 많아지면 시작 화면이 무거워질 수 있다.

대응:
- MVP에서는 지도자 3명만 제공
- 추천 태그를 달아 첫 선택을 쉽게 만든다

### 리스크 2: 숫자 중복
지도자 특성, 영웅 스킬, 경제 보너스가 겹치면 무엇이 효과를 냈는지 모호해질 수 있다.

대응:
- 결과 화면과 전투 텍스트에서 어떤 시스템이 발동했는지 분리해서 보여준다

### 리스크 3: 최적해 고착
한 지도자가 너무 좋으면 다른 지도자가 버려진다.

대응:
- 지도자별 강점 영역을 분명히 나눈다
- 특정 스테이지에서는 다른 지도자가 더 유리하게 만든다

## Design Principle

지도자 시스템의 핵심은 “더 강한 캐릭터 하나를 고르는 게임”이 아니라
“이번 전투를 어떤 철학으로 풀지 고르는 게임”이 되는 것이다.

그래서 지도자는 숫자보다 먼저 **방식**을 바꿔야 한다.
공격형 지도자는 빨리 끝내고 싶게 만들고, 수비형 지도자는 버티는 맛을 주고,
경제형 지도자는 계산과 성장의 재미를 줘야 한다.

플레이어가 지도자를 바꾸는 순간, 전투의 리듬과 생각의 순서가 달라져야 한다.
