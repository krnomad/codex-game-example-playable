## 1. Overview

바람의나라의 수묵화 픽셀아트 감성과 리니지클래식의 강화 이펙트 화려함을 결합한
32x32 한국 판타지 픽셀아트 스타일. 최근 프로토타입 기준으로는 여기에
`월광 전술실(Moonlit Command Room)` UI 톤을 더해, 전장 밖 HUD는 얇고 차가운 유리 패널처럼,
전장 안은 따뜻한 금빛 경고와 맥박으로 대비시키는 방향을 채택한다.

---

## 2. Player Fantasy

> 수묵화 속 고요한 산사에서 금빛 오라를 두른 영웅들이 요괴 군단을 막아선다.
> 평소에는 고전적이고 차분하지만, 강화된 영웅이 스킬을 쓰는 순간
> 화면이 화려한 이펙트로 가득 찬다.

**대비의 미학**: 배경은 차분한 수묵화풍 → 영웅/이펙트는 화려하고 빛남

**새 시각 축**: 전투 HUD는 "게임 UI"가 아니라 야전 지휘실. 한지/목재보다
밤하늘, 청람색 유리, 금빛 경보선, 달빛 스캔 라인이 중심이 된다.

**현재 프로토타입 적용 축**:
- 영웅 카드: 작은 텍스트 카드가 아니라 초상화 중심 전술 패널
- 지도자 선택: 아트 우선 대형 캐릭터 카드 + 초상 배지
- 보물 공방: 유물별 전용 일러스트와 단계 바를 가진 수집형 성장 화면
- 강화 결과: 성공/실패 순간을 전면 오버레이로 보여주는 의식적 연출

## 3. Detailed Rules

### 3.1 픽셀 해상도

- **캐릭터 스프라이트**: 32x32 (영웅, 적, NPC)
- **타일/배경**: 32x32 타일 기반
- **UI 아이콘**: 16x16 또는 32x32
- **게임 해상도**: 960x540 (16:9) → 브라우저에서 스케일링
- **스케일링**: 정수배 스케일링만 (2x, 3x) — 픽셀 깨짐 방지

### 3.2 컬러 팔레트

**배경 (수묵화 톤)**
- 주조: 먹색(#2C2C2C), 회백색(#E8E0D0), 담청(#6B8E9B)
- 산/나무: 짙은 녹(#3A5A40), 연녹(#8CB369)
- 건물/기와: 적갈색(#8B4513), 회색(#7A7A7A)

**영웅 (선명한 톤)**
- 무사: 붉은 갑옷(#C0392B), 금 장식(#F1C40F)
- 궁수: 녹색 의복(#27AE60), 갈색 활(#8B6914)
- 도사: 흰 도포(#ECF0F1), 청색 문양(#2980B9)
- 무녀: 분홍 치마(#E84393), 보라 상의(#6C3483)

**적 (어두운 톤)**
- 도깨비: 청록(#1ABC9C) + 붉은 눈
- 원귀: 흑자색(#4A235A) + 녹색 불꽃
- 대요괴: 짙은 적색(#922B21) + 금문양

**강화 이펙트 컬러**
- +1~+3: 흰색/은색(#FFFFFF, #C0C0C0)
- +4~+6: 청색(#3498DB, #2471A3)
- +7: 금색(#F1C40F, #F39C12)
- +8~+9: 적색/주황(#E74C3C, #E67E22)
- +10: 무지개 (금+청+자+홍 순환)

### 3.3 애니메이션 프레임 기준

| 동작 | 프레임 수 | 비고 |
|------|----------|------|
| Idle | 4 | 호흡/바람에 옷 흔들림 |
| Attack | 4~6 | 무기/마법 타입별 차이 |
| Skill | 6~8 | 화려한 이펙트 포함 |
| Hit | 2 | 짧은 타격 반응 |
| Death | 4 | 적 전용 |
| Walk (적) | 4 | 경로 이동 |
| 강화 오라 | 4 (루프) | 강화 단계별 별도 |

### 3.4 강화 시각 이펙트 상세

**+1~+3: 은은한 빛**
- 영웅 외곽에 1px 흰색 하이라이트가 2프레임 간격으로 깜빡임
- 파티클 없음, 미세한 변화

**+4~+6: 청색 오라**
- 영웅 주변 4px 범위에 반투명 청색 오라 (알파 40%)
- 느린 회전 (4프레임 루프)
- 바람의나라 마법 무기의 푸른빛 참조

**+7: 금빛 오라 (첫 번째 임계점)**
- 영웅 주변 6px 금빛 오라 (알파 50%)
- 작은 빛 파티클 2~3개가 위로 떠오름 (8프레임 루프)
- 배치 시 다른 영웅과 확연히 구분되는 시각적 존재감
- 리니지 +7 무기의 "빛나는 무기" 느낌 재현

**+8~+9: 화염 오라**
- 영웅 발 아래에서 붉은 기운이 피어오름
- 적색/주황 파티클이 지속 방출 (6프레임 루프)
- 위압감 있는 비주얼

**+10: 전설 (무지개 + 빛 방사)**
- 금색 → 청색 → 자색 → 홍색 순환 오라 (16프레임 루프)
- 빛줄기가 영웅 위로 솟아오름
- 이름 위에 별 3개 아이콘
- 화면 내 어디에 있든 즉시 눈에 들어오는 존재감

### 3.5 스킬 이펙트 방향

| 유형 | 스타일 | 참조 |
|------|--------|------|
| 근접(무사) | 칼기 잔상 + 붉은 베기 자국 | 사극 검격 |
| 원거리(궁수) | 빛나는 화살 궤적 + 관통 잔상 | 바람의나라 궁수 |
| 범위(도사) | 마법진 + 하늘에서 떨어지는 뇌전/화염 | 수묵 부적 문양 |
| 지원(무녀) | 치유 빛 파장 + 꽃잎 파티클 | 한국 무속 굿 |

### 3.6 UI 아트 스타일

- 프레임/테두리: 한국 전통 문양 (당초문, 연꽃)
- 버튼: 나무판 + 붓글씨 텍스트 느낌
- 강화 UI: 대장간 배경 + 모루 + 화로 — 어두운 톤에 불빛만 밝음
- HP 바: 한지 질감 위 먹물 게이지
- 폰트: 궁서/명조 계열 픽셀 폰트 (한글 가독성 우선)

### 3.7 월광 전술실 HUD 방향

- 상단 HUD: 얇은 유리 칩이 좌상단부터 정렬된 지휘 정보 띠
- 우측 패널: 영웅/버튼이 세로로 정렬된 전술 도구함
- 지도자 패널: 차가운 청람색 톤, XP 바와 재능 포인트 강조
- 보물 공방: 전투 화면 위에 떠오르는 비밀 공방 같은 자색/금색 계열 모달
- 결과 화면: 단순 점수판이 아니라 전투 후 보고서

색과 재질 규칙:
- 차가운 정보 레이어: 청람색, 유백색, 어두운 남색 유리
- 즉시 행동 레이어: 금색, 호박색, 붉은 경고
- 메타 성장 레이어: 자색, 분홍빛, 금색 하이라이트

추가 시각 규칙:
- 지도자 패널은 "상태 위젯"이 아니라 초상과 XP가 함께 보이는 지휘 카드로 읽혀야 한다
- 보물 공방 카드는 아이콘보다 일러스트 비중이 높아야 한다
- 영웅/유물/지도자의 이미지는 동일한 조명 언어를 공유해야 한다
- UI는 텍스트 설명보다 이미지, 색, 배지, 단계 바로 먼저 정보를 전달해야 한다

### 3.8 모션 톤

- 전장 경로: 웨이브 진행 시 길 위에 맥박처럼 흐르는 빛
- 빈 슬롯: 배치 가능 상태에서 잔잔한 호흡
- 지도자/공방 알림: 짧고 강한 배너형 등장
- 엘리트 적: 일반 적보다 확실히 읽히는 고리형 강조
- 강화 성공: 화면 중앙에 유물 일러스트가 떠오르며 고리/플래시가 확장되는 짧은 승리 연출
- 강화 실패: 작은 흔들림과 자색 균열 톤으로 실패를 즉시 인지시키되, 벌점보다 "다음 시도 압박"을 느끼게 한다

원칙:
- 상시 미세 진동보다 상태 변화 시 존재감을 준다
- 모션은 시선 유도용이지 장식용이 아니다
- 성공 연출은 "보상 확인", 실패 연출은 "리스크 인지"로 역할이 달라야 한다

## 4. Formulas

### 4.1 스프라이트 메모리 예산

```
영웅 1종 = (idle 4 + atk 6 + skill 8 + hit 2) × 32×32 × 4byte = ~10KB
영웅 8종 = ~80KB
오라 이펙트 4종 × 8프레임 = ~8KB
적 6종 = ~48KB
UI 에셋 = ~64KB
배경 타일 = ~128KB
총 추정: ~328KB (256MB 메모리 예산의 0.13%)
```

### 4.2 파티클 버짓

```
최대 동시 파티클 = 200개 (60fps 유지 기준)
영웅 1기 오라 파티클 ≤ 5개
스킬 이펙트 파티클 ≤ 20개
동시 배치 영웅 8기 × 5 = 40 파티클 (오라)
동시 스킬 2개 × 20 = 40 파티클 (스킬)
여유: 120 파티클 (적 처치 이펙트 등)
```

## 5. Edge Cases

| 상황 | 처리 |
|------|------|
| 저사양 브라우저에서 파티클 과다 | 파티클 수를 절반으로 줄이는 "저사양 모드" 옵션 |
| 여러 +7 이상 영웅 동시 배치 | 오라 겹침 시 가장 높은 단계 오라만 전체 발광, 나머지는 축소 |
| 모바일 브라우저 작은 화면 | 32x32 → 2x 스케일 = 64x64 표시. UI 터치 영역 최소 44px |
| 컬러블라인드 유저 | 강화 단계를 색상만이 아닌 아이콘/숫자로도 표시 |

## 7. Tuning Knobs

| 파라미터 | 현재 값 | 안전 범위 | 영향 |
|---------|--------|----------|------|
| `max_particles` | 200 | 50~500 | 낮추면 성능 개선, 높이면 화려함 증가 |
| `aura_alpha` | 0.4~0.5 | 0.2~0.8 | 높이면 오라 강조, 낮추면 은은함 |
| `particle_scale_low` | 0.5x | 0.25~1.0 | 저사양 모드 파티클 크기 |
| `skill_effect_duration` | 0.5s | 0.3~1.0s | 짧으면 빠른 진행, 길면 화려함 |

## 8. Acceptance Criteria

### 해상도 & 스케일링
- [ ] 모든 스프라이트가 32x32 기준으로 제작되었다
- [ ] 2x, 3x 정수배 스케일링 시 픽셀이 깨지지 않는다
- [ ] 960x540 기준 해상도에서 UI 요소가 겹치지 않는다

### 컬러 & 분위기
- [ ] 배경이 수묵화 톤(먹색, 담청, 회백)으로 통일되었다
- [ ] 영웅 스프라이트가 배경 위에서 선명하게 구분된다
- [ ] 강화 이펙트 컬러가 단계별로 명확히 구분된다

### 강화 이펙트
- [ ] +1~+3 영웅에 흰빛 테두리가 표시된다
- [ ] +4~+6 영웅에 청색 오라가 표시된다
- [ ] +7 영웅에 금빛 오라 + 파티클이 표시되며, 다른 영웅과 확연히 구분된다
- [ ] +8~+9 영웅에 화염 오라가 표시된다
- [ ] +10 영웅에 무지개 순환 오라 + 빛 방사가 표시된다

### 성능
- [ ] 영웅 8기 동시 배치 + 오라 시 60fps를 유지한다
- [ ] 동시 스킬 2개 발동 시 60fps를 유지한다
- [ ] 저사양 모드에서 파티클 50% 감소가 적용된다

### 접근성
- [ ] 강화 단계가 색상 외 숫자/아이콘으로도 표시된다
- [ ] 모바일에서 터치 영역이 최소 44px이다
